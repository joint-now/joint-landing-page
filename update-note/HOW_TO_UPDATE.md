# 업데이트 노트 갱신 방법 (로컬 파일 시스템)

이 프로젝트는 현재 **로컬 개발 모드**로 동작 중입니다.
서버는 사용자 컴퓨터의 특정 폴더를 직접 읽어와 웹사이트에 표시합니다.

---

## 📂 파일 위치

**경로**: `../joint-docs/products/update-notes/public`
(이 프로젝트 폴더의 바로 상위 폴더에 `joint-docs`가 있어야 합니다)

## 🚀 새 업데이트 노트 추가하기 (폴더 방식 추천)

업데이트 노트마다 이미지가 섞이지 않도록 **날짜별 폴더**를 만들어 관리하는 것을 추천합니다.

### 1. 폴더 생성
위 경로에 날짜와 제목으로 폴더를 만듭니다.
예: `2026-02-05-feature-update`

### 2. 파일 및 이미지 추가
그 폴더 안에 `.md` 파일과 이미지들을 넣습니다.

```text
public/
├── 2026-02-05-feature/
│   ├── update.md       (이름은 자유)
│   ├── thumbnail.png
│   └── screenshot.png
```

### 3. 파일 내용 작성
`.md` 파일 작성 시 같은 폴더에 있는 이미지를 바로 이름만 적으면 됩니다.

```markdown
---
title: "[Update] 26년 2월 5일 업데이트"
date: "2026-02-05"
tags: ["New", "Improvement"]
image: "thumbnail.png"
---

## 주요 기능
여기에 본문 내용을 작성하세요.

![스크린샷](screenshot.png)
```

> **참고**: 서버가 자동으로 `2026-02-05-feature/thumbnail.png` 처럼 경로를 찾아서 연결해줍니다.

## 🔄 사이트 반영

1. 파일을 저장합니다.
2. 웹사이트(`http://localhost:3001`)를 **새로고침**합니다.
3. 바로 반영됩니다! (서버 재시작 필요 없음)
